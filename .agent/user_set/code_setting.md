

# 代码说明

- 你在扫描学习项目代码时，只需要关注 src/main 文件夹下面的文件，项目工程的其他文件夹对你没有作用


# 执行代码任务

- 如果用户没有明确要求你改代码，你不需要改代码，询问用户即可
- 你修改完代码后不需要自己进行编译
- 代码必须尽量优雅


# 代码注释

- **注释要求**：
  - 写方法要写注释，要用中文，这个方法用来干什么的，如果代码中有多个步骤也需要增加注释更易于人类理解
  - 方法注释要写传参返回参数等是什么，步骤注释不要使用1.,2.,3.，因为使用1.,2.,3.,会导致添加步骤时还得调整序号
  - 方法统一使用文档注释，代码步骤只能用行内注释//，代码步骤注释后面不需要有句号，不允许注释在代码的尾部，比如重开一行，新建文件的时候作者使用“xxx”这个名字

# 日志打印

- 所有项目禁止使用`log.warn`，只可以只用`info`或者`error`级别日志
- **注释前缀规范** 在编写涉及较大模块、跨系统交互或完整业务链路的代码时，**所有相关的日志（Log）和关键注释必须统一添加双重方括号前缀**，以便于后续检索排查。
- **格式标准**：`【系统/业务方】【具体动作/任务】`
- **应用场景**：`log.info("【xxx】【xxx】开始组装请求报文，参数为: {}", request);`
